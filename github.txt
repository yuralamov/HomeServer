Начало работы с GitHub

Используя протокол SSH, вы можете подключаться и аутентифицироваться на удаленных серверах и сервисах. С помощью ключей SSH вы можете подключаться к GitHub без указания имени пользователя или пароля при каждом посещении.
Создание ssh
Проверка установленных ключей
ls -al ~/.ssh
Если ключ есть, то можно его использовать (перейти )

Сгенерировать новый ключ
ssh-keygen -t rsa -b 4096 -C "Ваше_мыло@на.github"
Утвердить имя (или изменить его) сгенерированного ключа
~/.ssh/id_rsa
Ввести парольную фразу
Rfrfz-njgfhjkmyfzahfpf
Получит отпечаток созданного ключа
The key fingerprint is:
SHA256:какая-то-галиматья и Ваше_мыло@на.github
The key's randomart image is:
+---[RSA 4096]----+
|     o.o+SB.     |
+----[SHA256]-----+

Запустите ssh-agent в фоновом режиме.
eval "$(ssh-agent -s)"

Добавьте свой закрытый ключ SSH в ssh-agent.
ssh-add ~/.ssh/id_rsa

Скопируйте ключ SSH в буфер обмена.
Либо через текстовый редактор или через xclip
sudo apt-get install xclip
xclip -sel clip < ~/.ssh/id_rsa.pub
Вставьте свой ключ в поле "Key".
Нажмите Add SSH key.
Если будет предложено, подтвердите свой пароль GitHub

Переключение удаленных URL-адресов с HTTPS на SSH и обратно
Откройте терминал и перейдите в рабочую дирректорию.
Если есть созданный репозиторий, то склонировать git
git@github.com:вашлогиннагитхобе/вашрепозиторий.git && cd вашрепозиторий
Вывод изменяемого репозитория
git remote -v
origin	git@github.com:вашлогиннагитхобе/вашрепозиторий.git (fetch)
origin	git@github.com:вашлогиннагитхобе/вашрепозиторий.git (push)
git remote set-url origin git@github.com:вашлогиннагитхобе/вашрепозиторий.git
Посмотрим, что получилось
git remote -v
origin	https://github.com/вашлогиннагитхобе/вашрепозиторий.git (fetch)
origin	https://github.com/вашлогиннагитхобе/вашрепозиторий.git (push)

Переключим канал обратно
git remote set-url origin https://github.com/вашлогиннагитхобе/вашрепозиторий.git
и т.д.

Проверим, что нам ответят в терминале
ssh -T git@github.com

Управление подтверждением подписи коммитов (GPG)
Проверить наличие пар GPG
gpg --list-secret-keys --keyid-format LONG

Сгенерируем новую пару ключей для подписи поделок GitHub
gpg --full-generate-key
Затем выведем публичный ключ на экран
gpg --list-secret-keys --keyid-format LONG
В строке sec   4096R/ следует идентификатор, который вводим в команду терминала
gpg --armor --export ??????????
Скопируйте ключ GPG, начиная с ----- BEGIN PGP PUBLIC KEY BLOCK ----- и заканчивая ----- END PGP PUBLIC KEY BLOCK -----
и вставить в GPG Key GitHub

Конфигурируем gpg ключ с github
git config --global user.signingkey тут этот же идентификатор
и добавим ключ в профиль
test -r ~/.bash_profile && echo 'export GPG_TTY=$(tty)' >> ~/.bash_profile
echo 'export GPG_TTY=$(tty)' >> ~/.profile
